# PTCGP Meta Intelligence Engine

**Live site:** https://abothegod.github.io/ptcgp-meta-engine/

A self-contained, single-file web app that tracks the Pokémon TCG Pocket competitive meta in real time — pulling live tournament data, simulating head-to-head matchups, and letting you build and test your own decks.

---

## Features

### Tier List
Decks ranked S / A / B / C by a composite score across six strategic dimensions:

| Dimension | Weight | What it measures |
|---|---|---|
| Win Rate | 40 % | Raw tournament win rate from Limitless standings |
| Meta Share | 20 % | Log-weighted popularity across the last 50 tournaments |
| Type Coherence | 10 % | Penalty for multi-energy requirements |
| Evolution Lines | 10 % | Full Stage-1 / Stage-2 chains present |
| Setup Speed | 10 % | Estimated turns to first meaningful attack |
| Disruption Value | 10 % | Red Card / Sabrina / stall potential |

Tier thresholds: S ≥ 75 · A ≥ 60 · B ≥ 45 · C < 45.
Toggle between **Sort by Tier** and **Sort by Win Rate** with the sort buttons above the filter bar.
Filter by energy type. Click any deck card to see the full 20-card decklist.

### Card Database
Browse every card released in PTCG Pocket. Sourced from
[flibustier/pokemon-tcg-pocket-database](https://github.com/flibustier/pokemon-tcg-pocket-database)
and patched into the HTML at each bi-weekly update.

- Search by name, filter by type / set / rarity / sort order
- 60-card paginated grid with CDN art thumbnails
- Click any card for full detail (attacks, abilities, energy costs, HP, retreat)
- **+ Add** button on each tile sends the card directly to the Deck Builder
- New-cards badge on the tab shows how many cards were added in the latest update

### Matchup Matrix
A heatmap of simulated head-to-head win rates between every tracked meta deck.
Generated by [deckgym-core](https://github.com/bcollazo/deckgym-core) (2,000 games per pairing)
during the bi-weekly pipeline run.
Color scale: red ≤ 40 % → yellow 50 % → green ≥ 60 %.
Click any cell or header to open that deck's full decklist.

### Custom Deck Builder
Build, save, and manage personal 20-card PTCG Pocket decks — all stored in `localStorage`.

- Search and filter the full card database inside the builder
- Live stats bar: card count, progress fill, per-type distribution
- Validation message when deck is incomplete or over the limit
- **Save** — persists to `localStorage` under key `ptcgp_decks`
- **Export** — copies or downloads the deck in DeckGym text format (`Card Name qty`)
- **Import** — paste any DeckGym-format text to populate the builder instantly
- **Edit / Delete** from the saved decks panel

### Simulate vs Meta
Test a saved deck against the full meta without writing any code.

1. Click **Simulate vs Meta** on any saved deck card.
2. The modal shows your deck in DeckGym format with copy / download options.
3. Click **Download simulate-deck.js** — the browser generates a self-contained
   Node.js script with the current meta snapshot embedded.
4. Run it locally: `node simulate-deck.js deck.txt`
5. Paste the output back into the modal to see a color-coded bar chart of your
   win rates against every meta deck.

If your deck ID already appears in `MATCHUP_MATRIX` (it's a meta deck), the
pre-computed results are shown immediately — no local run needed.

## How Updates Work

```
GitHub Actions (cron: 1st + 15th of month)
  │
  ├─ validate-card-db job
  │    └─ Fetch flibustier cards.json — verify > 400 entries (continue-on-error)
  │
  └─ update-meta job
       ├─ Install Rust + cache Cargo
       ├─ Clone / build deckgym-core
       ├─ node update-meta.js
       │    ├─ Phase 0  Fetch full card database (flibustier)
       │    ├─ Phase 1  Fetch last 50 standard-format tournaments (Limitless API)
       │    ├─ Phase 2  Merge with ptcgpocket.gg editorial tiers
       │    ├─ Phase 3  Enrich archetypes (energy types, evo lines, disruption)
       │    ├─ Phase 4  Score & assign tiers — no deck-count cap
       │    ├─ Phase 5  Build optimal 20-card decklists
       │    ├─ Phase 5b Run deckgym-core simulations (N×(N-1)/2 matchups, batched ×8)
       │    └─ Phase 6  Patch FULL_CARD_DB · CARD_REGISTRY · META_SNAPSHOT
       │                    · MATCHUP_MATRIX directly into ptcgp-meta-engine.html
       ├─ Validate output (file size, entry counts, JS syntax)
       └─ git commit + push
            "chore(meta): bi-weekly update YYYY-MM-DD — N decks · M cards · P matchups simulated"
```

## Running Locally

Requires **Node 18+**. No npm install — uses only Node built-ins and native `fetch`.

```bash
# Update meta data only (no simulations)
node update-meta.js

# Update with simulations (deckgym-core must be built first)
DECKGYM_PATH=./deckgym-core/target/release/deckgym node update-meta.js
```

The script overwrites `ptcgp-meta-engine.html` in place.

## Simulating Your Own Deck

1. **Install Rust** — https://rustup.rs

2. **Build deckgym-core:**
   ```bash
   git clone https://github.com/bcollazo/deckgym-core.git
   cd deckgym-core && cargo build --release
   ```

3. **Get your deck file** — open the My Decks tab, click a saved deck,
   then **Download deck.txt**.

4. **Get the helper script** — in the same modal, click **Download simulate-deck.js**.
   This script has the current meta snapshot embedded; no extra files needed.

5. **Run:**
   ```bash
   node simulate-deck.js deck.txt
   ```

6. **See results** — paste the terminal output into the Results textarea
   in the modal and click **Parse Results**.

## Data Sources & Credits

| Source | URL | What it provides | Update frequency |
|---|---|---|---|
| Limitless TCG API | https://play.limitlesstcg.com/api | Win rates + meta share from raw tournament standings | Real-time (pulled every update cycle) |
| ptcgpocket.gg | https://ptcgpocket.gg/tier-list/ | Editorial tier overrides | Pulled each update cycle |
| flibustier/pokemon-tcg-pocket-database | https://github.com/flibustier/pokemon-tcg-pocket-database | Full card metadata (name, type, HP, attacks, abilities, images) | Per-set releases |
| deckgym-core | https://github.com/bcollazo/deckgym-core | Rust simulation engine for head-to-head matchup data | Bi-weekly pipeline |

Win rates and meta share are computed from raw player records — not imported from any pre-ranked list.
A minimum of **10 appearances** across tournaments is required for an archetype to qualify.
Tournaments with fewer than **20 players** are excluded.

## Project Structure

```
ptcgp-meta-engine.html          Single-file frontend (HTML + inline CSS + inline JS)
                                  Patched constants: FULL_CARD_DB · CARD_REGISTRY
                                                     META_SNAPSHOT · MATCHUP_MATRIX

update-meta.js                  Data pipeline (Node 18+, no dependencies)
                                  Phase 0  fetchCardDatabase()     — flibustier JSON
                                  Phase 1  fetchLimitlessData()    — tournament standings
                                  Phase 2  mergeArchetypes()       — editorial overrides
                                  Phase 3  enrichArchetype()       — strategy engine
                                  Phase 4  scoreArchetype()        — 6-dimension score
                                  Phase 5  buildOptimalDeck()      — 20-card list
                                  Phase 5b runSimulations()        — deckgym-core
                                  Phase 6  patchHtml()             — write HTML

.github/workflows/
  update-meta.yml               Bi-weekly GitHub Actions workflow
                                  Jobs: validate-card-db → update-meta
                                  Inputs: dry_run · force_resimulate
```
